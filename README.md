# SpringBootSubject-Evaluation
Smart cloud IT 48기 과목평가 

**평가 일자:** 2025.12.26

## 1. 프로젝트 개요
본 프로젝트는 **Spring Boot**, **JPA**, **MySQL**을 이용하여 웹 애플리케이션을 구현하는 과제입니다. <br> 사용자는 시험에 응시하고, 시스템은 이를 채점하여 결과를 저장 및 조회할 수 있는 기능을 제공합니다.

### 🛠 기술 스택
- **Language:** Java
- **Framework:** Spring Boot
- **Database:** MySQL
- **ORM:** JPA (Hibernate)
- **Frontend:** HTML/CSS (Thymeleaf 권장), JavaScript
---

## 2. 제출 가이드라인

### 📦 제출 파일 구성
완성된 프로젝트는 아래 내용을 포함하여 압축(`이름.zip`) 후 메일로 제출합니다.
1. **프로젝트 소스 코드** (반드시 포함 여부 확인)
2. **SQL 파일** (`*.sql`)
    - 테이블 생성 (`CREATE`) 구문
    - 정답 데이터 입력 (`INSERT`) 구문
    - 테스트용 학생 응시 데이터 5개 (`INSERT`) 구문
3. **결과 보고서** (Word 또는 PDF)
    - 각 기능별 실행 화면 캡처 포함

### 🚫 채점 불가 사유
- DB 생성/테스트 데이터 SQL 또는 실행 화면 캡처 미포함 시
- 프로젝트 실행 불가 시
- `CREATE`, `INSERT` 구문 실행 시 테이블이나 데이터가 없어 실행이 안 되는 경우 **(0점)**

## 3. 데이터베이스(DB) 요구사항
*DB 이름, 계정, 비밀번호는 채점자가 수정하여 테스트하므로 설정 파일(`application.properties` 등) 관리에 유의합니다.*

### 필수 SQL 구문
1. **정답 테이블:** 시험 문제의 정답을 저장하는 테이블 생성 및 데이터 입력
2. **응시자 정보 테이블:** 응시자의 개인정보와 답안을 저장하는 테이블 생성
3. **샘플 데이터:** 테스트용 학생 응시 정보 5개 입력

## 4. 기능별 요구사항

### 1) 메인 화면
- **접근 권한:** 로그인/인증 없이 접근 가능
- **메뉴 구성:**
    - `[시험 응시]` 클릭 시 → 시험 응시 화면으로 이동
    - `[시험결과 조회]` 클릭 시 → 응시 결과 목록 화면으로 이동

### 2) 시험 응시 화면
#### 입력 Form
- **응시자 정보:** 이름, 이메일, 비밀번호 (필수 입력)
- **시험 문제:** (DB가 아닌 HTML 내에 하드코딩하여 작성)

#### 문제 목록 (Java)
1. **Java에서 문자열(String) 내용 비교로 올바른 것은?** (객관식)
2. **오버로딩에 대한 설명으로 옳은 것은?** (객관식)
3. **배열 길이를 얻는 올바른 코드는?** (객관식)
4. **Java에서 클래스를 상속받을 때 사용하는 키워드는?** (주관식)
5. **예외가 발생할 가능성이 있는 코드를 감싸서 예외를 처리할 때 사용하는 키워드는?** (주관식)

#### 유효성 검사 (JavaScript)
- 이름, 이메일, 비밀번호, 모든 문제의 답안은 **필수 입력 항목**입니다.
- 하나라도 입력되지 않은 상태로 "제출" 버튼 클릭 시 `alert` 메시지 출력 후 전송 중단.

### 3) 제출 처리 (채점 및 저장)
- **채점 로직:**
    - 서버(Service 계층)에서 정답 테이블을 조회하여 채점 수행.
    - 배점: 1문항당 20점 (총 100점 만점).
- **중복 검사:**
    - 이미 응시한 이메일일 경우 저장하지 않음.
    - "이미 응시한 이메일입니다. [이메일주소]" 오류 메시지(alert 등) 출력.
    - **중요:** 시험 화면으로 돌아왔을 때 기존 입력값(이름, 이메일, 선택 답안)이 폼에 유지되어야 함.
- **저장:** 채점 결과(점수), 제출 답안, 개인정보를 DB에 저장.

### 4) 시험 결과 보기
저장된 모든 응시 결과를 목록 형태로 출력합니다.

#### 출력 컬럼
`이름` | `이메일` | `1번` | `2번` | `3번` | `4번` | `5번` | `점수` | `응시일` | `결과(Pass/Fail)` | `삭제`

#### 정렬 기능 (메뉴 클릭 시 동작)
- **응시일순:** 날짜 오름차순 (Default)
- **이름순:** 이름 오름차순
- **성적순:** 성적 내림차순

#### 삭제 버튼
- 목록의 'X' 버튼 클릭 시 삭제 화면으로 이동.

### 5) 응시 기록 삭제
- **화면 구성:** 선택한 응시자의 이름, 이메일, 응시 날짜를 보여주고 **비밀번호**를 입력받음.
- **처리 로직:**
    - 비밀번호 일치 시: 해당 응시 정보 삭제 후 목록 화면으로 이동.
    - 비밀번호 불일치 시: 삭제하지 않음.
